# Day 05 - AI Assisted Terraform Review

## Objective

Learn how AI can help review Infrastructure as Code.

## Key Concepts

### Security
- Open security groups
- Overly permissive IAM roles
- Public resources

### Cost
- Oversized instances
- Unused resources
- Missing autoscaling

### Reliability
- Multi-AZ design
- Backup strategy
- Failover considerations

### Observability
- CloudWatch monitoring
- Logging
- Alerting

## Key Learning

AI is useful for reviewing Terraform code from:
- Security perspective
- Cost perspective
- Reliability perspective
- Observability perspective

## Important Rule

AI reviews infrastructure.

Engineers approve infrastructure.

## Daily Summary

Today I learned how to use AI as a Terraform review assistant instead of only a code generation tool.
