# Day 09 - AI Pair Programming for DevOps Engineers

# Objective

Learn how to use AI as a pair programmer to improve productivity, code quality, infrastructure automation, and engineering decision-making.

The goal is not to replace engineering skills but to accelerate development while maintaining production safety.

---

# What is AI Pair Programming?

AI Pair Programming is the practice of collaborating with AI during software development and infrastructure automation.

Instead of writing everything manually, AI assists by:

* Explaining code
* Generating boilerplate
* Reviewing infrastructure
* Suggesting improvements
* Creating documentation

The engineer remains responsible for validation and deployment.

# Engineering Principles

Never:

* Copy AI output blindly
* Deploy without review
* Ignore security concerns

Always:

* Review
* Test
* Validate
* Verify

---

# Questions to Ask AI

* Why?
* What assumptions are you making?
* What are the risks?
* How should I verify this?
* Is there a better approach?

---

# Key Learnings

* AI improves productivity.
* AI reduces repetitive work.
* AI accelerates learning.
* AI cannot replace engineering judgment.
* Verification is mandatory.
